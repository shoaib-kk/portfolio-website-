Exploratory analysis is not a contest to produce the most charts. It is a process for reducing uncertainty about a dataset and the problem around it.

## Start with decisions

Before plotting, write down what somebody might decide differently after seeing the analysis. That turns visualization into evidence rather than decoration.

Useful early questions include:

- What does one row represent?
- Which fields are measured and which are derived?
- Where are missing values concentrated?
- Which distributions make averages misleading?
- What cannot be concluded from this data?

<Callout title="A useful stopping point">
Stop adding charts when the next visualization does not change the explanation, reveal a limitation, or support a decision.
</Callout>

## Make limitations visible

A strong analysis states what the data cannot support. Sample bias, missing periods, inconsistent collection, and proxy variables belong in the main narrative—not in a footnote added at the end.
